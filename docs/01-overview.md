# DevLog Automator — 개요

> 최종 갱신: 2026-04-10

## 한 줄 정의

Claude Code 세션 로그를 자동 수집하고, Claude Code CLI로 구조화한 뒤, 인터랙티브 질문으로 맥락을 보완하여 데브로그 마크다운 파일을 생성하는 CLI 도구.

## 목적

업무 중 마주치는 **문제 → 원인 분석 → 판단 → 해결 → 결과**의 과정을 기록하고 싶지만, 실제로 일하면서 이를 정리할 시간이 없다.

이 도구는 이미 존재하는 Claude Code 대화 로그와 git 이력을 자동으로 수집·분석하여 초안을 만들어주고, 사용자는 핵심 맥락만 보완하면 되도록 한다.

**핵심 가치:**
- 성과 기록 — 내가 오늘 뭘 했는지 구조적으로 남긴다
- 회고 — 문제를 어떻게 풀었는지 돌아본다
- 성장 추적 — 시간이 지나며 판단력이 어떻게 바뀌는지 확인한다

## 사용자

- 범용 (Claude Code를 사용하는 모든 개발자)
- 초기 설정(`devlog init`)으로 저장 경로를 지정

## 실행 방식

| 항목 | 결정 |
|------|------|
| 도구 형태 | npm 글로벌 패키지 |
| 실행 명령 | `devlog` |
| 실행 시점 | 하루 1회, 퇴근 전 |
| 실행 환경 | 터미널 (어디서든, repo 안 찾아가지 않아도 됨) |

## 기술 스택

| 항목 | 선택 | 이유 |
|------|------|------|
| 언어 | TypeScript (strict) | npm 생태계, 타입 안전성 |
| 실행 | npm 글로벌 패키지 (`devlog`) | 어디서든 실행 가능 |
| AI | Claude Code CLI (`claude -p`) | 별도 API 키 불필요, 기존 인증 사용 |
| CLI 인터랙션 | @clack/prompts | 질문-답변, spinner, 색상 출력 내장 |
| 빌드 | tsup | 빠른 번들링 |
| Linter / Formatter | Biome v2 | ESLint+Prettier 대체 |

## 설계 원칙

1. **최소 입력** — 로그에서 읽을 수 있는 건 AI가 채우고, 맥락이 필요한 것만 질문
2. **적극적 질문** — 모호한 부분은 추측보다 질문 (태스크당 최대 5개)
3. **수정 가능** — 저장 전 초안 미리보기 + 프롬프트로 수정 가능
4. **배운 점은 본인 확인** — AI가 제안 후 사용자가 동의
5. **이중 저장** — 시간순 RAW 일지 + 프로젝트별 태스크 일지
6. **점진적 개선** — 처음엔 단순하게, 나중에 태그/분류 등 확장
