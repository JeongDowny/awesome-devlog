# 개발 단계

## 전체 파이프라인

```
$ devlog

[1단계: 선택]
├─ 최근 7일 중 날짜 선택 (multiselect)
├─ 날짜 범위로 세션 인덱싱 (history.jsonl 1회 읽기)
├─ 프로젝트 선택 (multiselect)
└─ 세션 세부 선택 (multiselect)

[2단계: 수집]
├─ 세션 로그 수집 (user/assistant 텍스트만, 60K자 제한)
└─ git log 수집 (프로젝트별 병렬, 날짜 범위)

[3단계: 구조화 (Claude Code CLI)]
├─ 세션 대화 + git log 분석
├─ 프로젝트별, 태스크 단위로 분리
├─ 문제 → 원인 → 해결 → 결과 초안 생성
└─ 모호한 부분에 [질문 필요: ...] 태그 삽입

[4단계: 인터랙티브 보완]
├─ Claude Code 외 작업 추가
├─ [질문 필요: ...] 태그 기반 질문 (태스크당 최대 5개)
├─ 배운 점 확인 (사용자 → AI 제안 → 동의)
└─ 초안 미리보기 + 프롬프트 수정 루프

[5단계: 저장]
├─ 시간순 RAW 일지 → <outputDir>/daily/YYYY-MM-DD_HHmm.md
├─ 프로젝트별 태스크 매칭 (Claude가 기존 파일과 비교)
└─ 프로젝트별 일지 → <outputDir>/projects/<프로젝트>/<태스크>.md
```

## 1단계 — 날짜·프로젝트·세션 선택

- 최근 7일 날짜를 multiselect로 표시 (오늘 기본 선택, 요일 라벨)
- 선택된 날짜의 최소~최대 범위로 `indexSessionsInRange` 1회 호출
- 불연속 날짜 선택 시 후필터링으로 해당 날짜만 추출
- 프로젝트 multiselect (워크트리 → 메인 병합 후 표시, 세션 수 포함)
- 선택된 프로젝트의 세션만 multiselect (전체 pre-selected)

## 2단계 — 로그 수집

- 프로젝트별 git log 병렬 수집 (날짜 범위 기반)
- 세션 로그 순차 수집 (I/O 바운드)
- `projectPathToDir`로 경로 변환 시 `/`와 `_`를 `-`로 치환
- 레거시 규칙(`/`만 치환) fallback 지원

## 3단계 — Claude Code CLI로 구조화

- `claude -p --output-format text`로 비대화형 호출
- Claude Code의 기존 인증 사용 (별도 API 키 불필요)
- 모호한 필드에 `[질문 필요: 구체적 질문]` 태그 삽입 지시
- 전체 프롬프트 300,000자 초과 시 데이터 잘림
- 타임아웃: 5분

## 4단계 — 인터랙티브 질문으로 보완

- Claude Code 외 작업 추가 (쉼표 구분 → 태스크 추가)
- 모든 태스크의 4개 필드(문제/원인/해결/결과)에서 `[질문 필요: ...]` 파싱
- 태스크당 최대 5개 질문, 필드 라벨 표시
- 답변 시 태그를 답변으로 대체, 건너뛰면 태그 제거
- 배운 점: 사용자 직접 입력 → 미입력 시 AI 제안 → 태스크별 동의 확인
- 초안 미리보기 후 저장/수정/취소 선택
- 수정 선택 시 프롬프트 입력 → Claude가 마크다운 수정 → 다시 미리보기 (반복 가능)

## 5단계 — 이중 저장

- **시간순 RAW 일지**: `<outputDir>/daily/YYYY-MM-DD_HHmm.md`
  - 메타데이터 헤더: 반영 날짜, 프로젝트별 작업시간, 작성 시각
  - 파일명에 시각 포함 → 같은 날 여러 번 실행해도 덮어쓰지 않음
- **프로젝트별 일지**: `<outputDir>/projects/<프로젝트명>/<태스크제목>.md`
  - Claude가 기존 파일 목록과 새 태스크를 비교하여 매칭 판단
  - 같은 작업이면 기존 파일에 날짜별 append (--- 구분선 + ## 날짜)
  - 새 작업이면 새 파일 생성
  - "기타" 프로젝트는 프로젝트별 저장 제외
