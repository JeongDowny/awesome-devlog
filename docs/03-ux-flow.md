# UX 흐름 — 인터랙티브 질문 + 저장

## 전체 흐름에서의 위치

```
[1단계: 날짜·프로젝트·세션 선택]
├─ 최근 7일 중 날짜 선택 (multiselect, 오늘 기본 선택)
├─ 프로젝트 선택 (multiselect, 전체 기본 선택)
└─ 세션 세부 선택 (multiselect, 선택 프로젝트 세션 전체 기본 선택)

[2단계: 수집 + 구조화]
├─ 세션 로그 + git log 수집
└─ Claude Code로 태스크 구조화

[3단계: 인터랙티브 보완] ← 이 문서
├─ Claude Code 외 작업 여부 질문
├─ 모호한 부분 질문 (태스크당 최대 5개, [질문 필요: ...] 태그 기반)
├─ 배운 점 확인 (사용자 → AI 제안 → 동의)
└─ 초안 미리보기 + 수정 프롬프트 루프

[4단계: 저장]
├─ <outputDir>/daily/YYYY-MM-DD_HHmm.md (시간순 RAW 일지)
└─ <outputDir>/projects/<프로젝트명>/<태스크제목>.md (프로젝트별 일지)
```

## 질문 설계 원칙

- 질문은 태스크당 최대 5개
- 한 줄 답변 수준으로 부담 없게 (건너뛰려면 Enter)
- 로그에서 읽을 수 있는 건 AI가 채우고, 모호한 부분만 질문
- 문제·원인·해결·결과 모든 필드에서 `[질문 필요: ...]` 태그 기반 질문 생성

## 질문 유형별 처리

| 유형 | 조건 | 질문 예시 |
|------|------|----------|
| Claude Code 외 작업 | 항상 (1회) | "Claude Code 외에 오늘 한 작업이 있어요?" |
| 맥락 보완 | `[질문 필요: ...]` 태그가 있는 필드 | "이 문제의 원인이 뭐였어요?" |
| 배운 점 | 항상 (1회) | "오늘 배운 점이 있어요?" |
| 배운 점 제안 | 사용자가 배운 점 없다고 답한 경우 | AI가 제안 → 동의 시 포함 |

## 저장 구조

### 시간순 RAW 일지

파일 경로: `<outputDir>/daily/YYYY-MM-DD_HHmm.md`

```markdown
# 2026-04-10

> **반영 날짜**: 2026-04-09, 2026-04-10
> **프로젝트**: 01_dnd (4h 32m), awesome-devlog (1h 15m)
> **작성 시각**: 2026-04-10 19:30

## 태스크 1. [태스크 제목]
**프로젝트**: [프로젝트명]
**시간**: HH:MM ~ HH:MM

### 문제 ...
### 원인 ...
### 해결 ...
### 결과 ...
### 배운 점 ...

---

## 태스크 2. ...
```

### 프로젝트별 일지

파일 경로: `<outputDir>/projects/<프로젝트명>/<태스크제목>.md`

```markdown
# CSS 호환성 이슈 수정

## 2026-04-09
**시간**: 14:00 ~ 16:30

### 문제 ...
### 해결 ...

---

## 2026-04-10
**시간**: 10:00 ~ 12:00

### 문제 ...
### 해결 ...
```

- 같은 태스크면 기존 파일에 날짜별로 append (Claude가 매칭 판단)
- 새 태스크면 새 파일 생성
- "배운 점"은 사용자 확인 후에만 포함
- Obsidian frontmatter/태그는 현재 도입하지 않음 (순수 마크다운)
- 작업 없는 날은 파일을 생성하지 않고 스킵
